ðŸ”¹ 2. Why PriorityQueue?
-------------------------
-> Normally, a Queue follows FIFO (First In, First Out) order.
-> But sometimes you need to process elements based on priority, not on the order they were inserted.
For example:
 -> In a task scheduler, high-priority tasks must execute first.
 -> In Dijkstraâ€™s shortest path, smallest distances must be processed first.
 -> Thatâ€™s where PriorityQueue is used.

 ðŸ”¹ 3. When to Use PriorityQueue?
----------------------------------
Use PriorityQueue when:
 -> You need to process elements by priority (not insertion order).
 -> You need an efficient way to access the smallest or largest element.


ðŸ”¹ 4. How PriorityQueue Works
-------------------------------
 -> Internally, it uses a binary heap (min-heap by default).
 -> The head of the queue is the smallest element (natural ordering) or according to a custom 
    Comparator you provide.
 -> Null elements are not allowed.
 -> Duplicate elements are allowed.
