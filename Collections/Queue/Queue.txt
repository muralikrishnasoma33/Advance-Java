üîπ 2. Why Queue?
-------------------
 -> We use a Queue when:We need to store data temporarily before using it.
 -> We want to process elements in the same order they were inserted (FIFO).
 -> We need a waiting line for tasks ‚Äî like printer jobs, requests, or customer service tickets.
 -> We want to implement schedulers, buffers, or messaging systems.
   ‚úÖ Purpose:
    To hold elements before processing them in an ordered way.

üîπ 3. When to Use Queue
-------------------------
Use a Queue when:
 -> You want elements to be processed one by one in insertion order.
 -> You‚Äôre working on task scheduling, job management, or data buffering.
 -> You‚Äôre implementing producer-consumer or multithreading systems.
üí° Example:
 -> Job queues in an operating system.
 -> Printing documents (first document prints first).
 -> Sending and receiving messages.

üîπ 4. How Queue Works
----------------------
Queue works in FIFO order:
‚û°Ô∏è Add (insert) ‚Üí from the rear (tail)
‚û°Ô∏è Remove (process) ‚Üí from the front (head)

üîπ 5. Key Points
------------------
 -> Queue follows FIFO order (First In, First Out).
 -> Queue allows duplicates ‚úÖ.
 -> Queue may allow null (depends on implementation).
 -> No direct method like get(index) to access elements by position ‚ùå.
 Common implementations:
    - LinkedList
    - PriorityQueue
    - ArrayDeque
