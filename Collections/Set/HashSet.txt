Definition :
--------------
   A Set in Java is a collection that does not allow duplicate elements.
-> It is part of the java.util package and is an interface that extends the Collection interface.

Syntax :
----------
   --> Set<Type> setName = new HashSet<>();

üí° Why use Set?
-----------------
We use a Set when:
  --> You want to store unique elements (no duplicates allowed).
  --> The order of elements doesn‚Äôt matter (depending on implementation).
  --> You want fast search, insertion, and deletion (especially with HashSet).
Example:
If you want to store student roll numbers where no number repeats, a Set is perfect.

üïì When to use Set?
---------------------
Use a Set when:
  --> You must ensure uniqueness (like IDs, usernames, etc.).
  --> You don‚Äôt care about insertion order (use HashSet).
  --> You need sorted data (use TreeSet).
  --> You want to maintain insertion order (use LinkedHashSet).

‚öôÔ∏è How Set works
------------------
Internally, most Sets use a hash table (like HashSet).
    > When you add an element:
It computes a hashcode for the element.
Checks if that hash already exists in the table.
If not, the element is added; otherwise, it‚Äôs ignored (no duplicates).

                              1. HashSet
===================================================================================================                               

1Ô∏è‚É£ Definition :
------------------
1. A HashSet in Java is a collection class that implements the Set interface.
2. It stores unique elements only and does not maintain any order of elements.
        -> Part of java.util package.
        -> Internally backed by a HashMap.                

Syntax :
----------
         --> HashSet<Type> set = new HashSet<>();

2Ô∏è‚É£ Why use HashSet? :
----------------------
We use HashSet because:
    --> We want unique elements only (no duplicates allowed).
    --> We need fast insertion, deletion, and search (average O(1) time).
    --> We don‚Äôt care about element order.
    --> Good for checking existence quickly.   

3Ô∏è‚É£ When to use HashSet :
-------------------------
Use HashSet when:
    --> Uniqueness of elements is required.
    --> Order of elements does not matter.
    --> You need fast operations (add, remove, contains).
    --> You are okay with allowing one null element.    

4Ô∏è‚É£ How HashSet works :
------------------------
--> Add element: HashSet uses hashCode() to determine the bucket.
     If bucket empty ‚Üí add element.
     If bucket occupied ‚Üí checks equals() to avoid duplicates.
--> Contains element: Computes hash ‚Üí checks bucket ‚Üí compares with equals().
--> Remove element: Computes hash ‚Üí finds bucket ‚Üí removes element.

‚úÖ Summary
--------------
Definition: HashSet = Set + HashMap, unique elements, unordered.
