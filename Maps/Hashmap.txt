1. HASHMAP â€” IN DEPTH DEFINITION :
===================================
HashMap is a keyâ€“value based data structure in Java that:

* stores data in buckets using hashing
* allows one null key and multiple null values
* does NOT maintain any order
* does NOT allow duplicate keys
* gives **O(1)** average time for insertion, deletion and lookup
* internally uses **array + linked list + red-black tree (from Java 8)** to store values efficiently

Internal core idea :
--------------------
HashMap = Hashing + Buckets + Key-Value Pairs

Keys are converted to bucket index using this formula:

index = hashcode(key) & (capacity - 1)

4. INTERNAL WORKING (in depth but simple)  :
==========================================
Step 1 â†’ `put(key, value)`
--------------------------
1. hashcode of key is generated
2. index = hashcode & (capacity - 1)
3. HashMap goes to that **bucket**
4. If bucket empty â†’ add new node
5. If bucket has nodes â†’ collision happens
6. Collision resolved by:

   * LinkedList (Java 7)
   * LinkedList + Red-Black Tree (Java 8)

**Treeing happens if a bucket has â‰¥ 8 nodes.**

Step 2 â†’ `get(key)`
---------------------
1. hashcode() of key
2. find bucket
3. compare keys using **equals()**
4. Return the value

Step 3 â†’ Rehashing
--------------------
When size exceeds:
capacity * loadFactor

New array created (double size)
All entries redistributed.

Default loadFactor = **0.75**

    public static void main(String[] args) {
