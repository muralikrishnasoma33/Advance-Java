‚úÖ 1. What is TreeMap?
------------------------
**TreeMap** is a class in Java that **stores key‚Äìvalue pairs in sorted order** (ascending order of keys).
It uses a **Red-Black Tree** internally.
‚úî Keys sorted automatically
‚úî No duplicates
‚úî Stores elements in **logarithmic time (O(log n))**

‚úÖ 2. Why TreeMap?
-------------------
Use TreeMap when you want a **Map + Sorting**.
Example:
* You want names in alphabetical order
* You want numbers sorted automatically
* You want a map where you can find **next higher key, lower key** etc.
HashMap does **NOT** maintain order.
LinkedHashMap preserves **insertion order**.
TreeMap preserves **sorted order**.

‚úÖ 3. How TreeMap Works Internally?
-----------------------------------
TreeMap stores entries in a **Red-Black Tree**.
So operations:
* insertion
* deletion
* search
take **O(log n)** time.
Keys must be:
* **Comparable** (String, Integer, etc)
  OR
* You must pass a **Comparator**

‚úÖ 4. When to Use TreeMap?
---------------------------
Use TreeMap when:
* You need **sorted keys**
* You need **range operations**
  * firstKey()
  * lastKey()
  * higherKey()
  * lowerKey()
* You need a **NavigableMap**

Don't use it when:
* You want fastest performance ‚Üí use HashMap
* You want insertion order ‚Üí LinkedHashMap

‚≠ê 5. TreeMap Implements What?
-------------------------------

| Interface        | Meaning                                                  |
| ---------------- | -------------------------------------------------------- |
| **Map**          | Key-value structure                                      |
| **SortedMap**    | Keys sorted                                              |
| **NavigableMap** | Extra navigation methods (higher, lower, floor, ceiling) |

SortedMap:
=============
üìò SortedMap ‚Äî Definition (for comparison)

SortedMap is an interface that represents a Map whose keys are always sorted in ascending order.
It defines methods to retrieve the first key, last key, sub ranges, etc., but does not define how sorting is implemented.

TreeMap is the main class that implements SortedMap.

üîπ SortedMap Methods
---------------------
| **Method**                   | **Description (Simple)**                                        |
| ---------------------------- | --------------------------------------------------------------- |
| `comparator()`               | Returns the comparator used for sorting (null = natural order). |
| `firstKey()`                 | Returns the first (smallest) key.                               |
| `lastKey()`                  | Returns the last (largest) key.                                 |
| `headMap(K toKey)`           | Returns keys `< toKey`.                                         |
| `tailMap(K fromKey)`         | Returns keys `>= fromKey`.                                      |
| `subMap(K fromKey, K toKey)` | Returns keys in range `[fromKey, toKey)`.                       |


NavigableMap :
==============
üìò NavigableMap ‚Äî Definition (for comparison)

NavigableMap is an extension of SortedMap that supports navigation through keys.
It provides methods to find the nearest keys to a given key, such as:

higher

lower

ceiling

floor

It also provides descending views and polling (remove-first/last) methods.
TreeMap is the main class that implements NavigableMap.

üîπ NavigableMap Methods (extra powerful)
-----------------------------------------

| **Method**                                                                 | **Description (Simple)**                                |
| -------------------------------------------------------------------------- | ------------------------------------------------------- |
| **lowerKey(K key)**                                                        | Returns greatest key **< key**                          |
| **floorKey(K key)**                                                        | Returns greatest key **‚â§ key**                          |
| **ceilingKey(K key)**                                                      | Returns smallest key **‚â• key**                          |
| **higherKey(K key)**                                                       | Returns smallest key **> key**                          |
| **lowerEntry(K key)**                                                      | Returns entry with greatest key < key                   |
| **floorEntry(K key)**                                                      | Returns entry with greatest key ‚â§ key                   |
| **ceilingEntry(K key)**                                                    | Returns entry with smallest key ‚â• key                   |

| **firstEntry()**                                                           | Returns entry with smallest key                         |
| **lastEntry()**                                                            | Returns entry with largest key                          |
| **pollFirstEntry()**                                                       | Removes and returns first entry                         |
| **pollLastEntry()**                                                        | Removes and returns last entry                          |
| **descendingMap()**                                                        | Returns a reverse-sorted map view                       |

‚≠ê 7. Simple Program (Very Easy)
----------------------------------
```java
import java.util.*;

class Demo {
    public static void main(String[] args) {

        TreeMap<Integer, String> tm = new TreeMap<>();

        tm.put(20, "Mahidhar");
        tm.put(10, "KR");
        tm.put(30, "Vibes");
        tm.put(25, "Java");

        System.out.println("TreeMap : " + tm);
        System.out.println("First Key : " + tm.firstKey());
        System.out.println("Last Key : " + tm.lastKey());
        System.out.println("Higher Key of 20 : " + tm.higherKey(20));
        System.out.println("Lower Key of 20 : " + tm.lowerKey(20));
    }
}

### **Output**

```
TreeMap : {10=KR, 20=Mahidhar, 25=Java, 30=Vibes}
First Key : 10
Last Key : 30
Higher Key of 20 : 25
Lower Key of 20 : 10
```

‚≠ê 9. Is there any big difference between TreeMap, SortedMap, NavigableMap?
-----------------------------------------------------------------------------
No.
TreeMap **is a concrete class**.
SortedMap and NavigableMap are **interfaces**.

TreeMap = SortedMap + NavigableMap.
